Introduction

Git is a free and open-source distributed version control system designed to handle everything from small to very large projects with speed and efficiency.

Key Features

  1. Strong support for non-linear development:Git supports rapid branching and merging and includes specific tools for visualizing and navigating a non-linear development history. A major assumption in Git is that a change will be merged more often than it is written.
  2. Distributed development:Git provides each developer a local copy of the entire development history, and changes are copied from one such repository to another. The changes can be merged in the same way as a locally developed branch very efficiently and effectively.
  3. Efficient handling of large projects: Git is very fast and scalable compared to other version control systems. The fetching power from a local repository is much faster than is possible with a remote server.
  4. Data Assurance:The Git history is stored in such a way that the ID of a particular version depends upon the complete development history leading up to that commit. Once published, it is not possible to change the old versions without them being noticed.
